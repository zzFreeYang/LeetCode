===================    Part1.基础知识  ===========================

1. 无状态对象一定是线程安全的。
    什么是无状态对象:既不含任何域，也不含任何对其他域的引用。计算过程中的临时状态仅仅存在线程栈上的局部变量中。两个线程访问了同一个Servlet，也不会相互影响。
    因此:大部分Servlet是无状态的，从而极大地降低了在实现Servlet线程安全性时的复杂性。只有当Servlet在处理请求时需保存一些信息，线程安全性才会成为一个问题。

2. 由于不恰当的执行时序而出现不正确的结果，是一个常见的问题，它有一个正式的名字: 竞态条件( Race Condition)
   常见的原因:“先检查后执行(Check-then-Act)”
   如何应对呢: 要避免竞态条件问题，必须在某个线程修改该变量的时候，通过某种方式防止其他线程使用这个变量。
       example : 使用原子变量类来确保这一点，参见Demo1_1
